# Tour vault (demo)

This folder is a **fully populated example** of what your research wiki looks like after ingest and build. It uses three real papers from the Chandrasekharan lab (UIUC):

| Paper | Venue | PDF |
|-------|-------|-----|
| [Does Positive Reinforcement Work?](wiki/papers/positive-reinforcement-reddit.md) | CHI 2025 | `raw/papers/chi2025-positive-reinforcement.pdf` |
| [The Language of Approval](wiki/papers/language-of-approval.md) | CHI 2026 | `raw/papers/chi2026-language-of-approval-2.pdf` |
| [r/popular Feed Audit](wiki/papers/popular-feed-audit.md) | ICWSM 2026 | `raw/papers/2502.20491v3.pdf` |

## How to browse

1. Open **`examples/minimal-vault`** as an Obsidian vault (or browse in any Markdown editor).
2. Start at **[index.md](index.md)** → **[wiki/overview.md](wiki/overview.md)** → explore the graph.
3. Compare with the **blank template** at the repo root (empty `index.md`, no papers).

## What's inside

- **`raw/`** — immutable PDFs + notes. Default layout: `raw/notes/abstracts/` and
  `raw/notes/themes/` (underscore filenames). Folders are configurable in `builder/data.py`.
- **`wiki/papers/`**, **`wiki/themes/`** — generated by the portfolio builder.
- **`wiki/sources/`** — LLM-ingest-style summary pages (one per paper).
- **`wiki/syntheses/demo-reward-attention-thread.md`** — a filed cross-paper analysis.
- **`builder/data.py`** — the corpus table driving the builder.

## Rebuild this demo

From the repo root:

```bash
python3 builder/build.py --vault examples/minimal-vault
```

## Start your own vault

Use the blank template at the repo root, or scaffold a fresh copy:

```bash
python3 builder/new_vault.py ~/Desktop/my-research-wiki "My Lab"
```
